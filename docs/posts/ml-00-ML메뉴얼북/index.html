머신러닝 모델 생성 가이드라인

머신러닝
Author

minstar

Published

July 21, 2023

머신러닝 모델을 만들면서 참고할 내용들을 정리했다.
프로젝트를 구상하면서 혹은 진행하면서 참고할 수 있도록 구성하려고 노력했다. 이 글이 조금이라도 누군가에게 도움이 되길.

모델링 순서도

1. 밑그림 그리기

  1. 만들고 싶은 모델 구상하기
  2. 모델을 어떤식으로 배포할 것인가?
  3. 어떤 모델을 이용할 것인가? (지도/비지도 예측/분류)
  4. 어떻게 테스트 할 것인가?

2. 데이터 수집

  1. 데이터 수집 가능한지 알아보기. 이미 정리된 데이터 혹은 직접 수집할 데이터.
  2. 데이터를 수집하고 이용함에 있어 법률상에 문제가 없는지 확인하기.
  3. 데이터 수집하고 잘 정리하기
  4. 민감한 정보는 삭제하거나 비식별화하기
  5. train/test 세트로 분리하기

3. 탐색적 데이터 분석

  1. 원본을 보호하기 위해 복사된 데이터 이용하기.
  2. 각 피쳐의 특징을 정리하기.
    • 각 피쳐의 의미 살펴보기
    • 기술통계보기
    • 누락된 값, 노이즈 체크하기
    • 모델 작동 가능성 따져보기
    • 분포 형태 확인하기
  3. 지도학습이라면 피쳐/타겟 으로 분리하기
  4. 시각화하기
  5. 피쳐들간 상관관계 확인하기
  6. 모델 제작할때 참고하기 위해 위의 내용들 따로 빼두기.

4. 데이터 준비하기

  1. 원본을 보호하기 위해 복사된 데이터 이용하기.
  2. 데이터 전처리에 사용한 코드는 모두 함수화 하기.
    • 데이터 파이프라인 제작을 위해서
    • 한번 해두면 편함
  3. 데이터 전처리
    • 이상치 수정 삭제
    • 결측치 수정 삭제
  4. 특성 선택 (선택사항)
  5. 적절하게 특성 조작하기
    • 연속 특성 이산화하기
    • 특성 분해하기 (범주형, 날짜/시간)
    • 특성을 새로운 특성으로 바꿔주기 (요일추가)
  6. 특성들 스케일 조정하기
    • 표준화, 정규화

5. 모델 선택하기

  1. 여러 종류의 모델을 기본 매개변수만 사용해서 빠르게 많이 훈련시켜보기. (적합한 모델 찾기 위함)
  2. 성능을 측정하고 비교하기
    • 각 모델에서 N-겹 교차 검증을 사용해 N개의 폴드에 대해 성능에 대한 평균과 표준편차 구하기.
  3. 각 알고리즘에서 가장 두드러진 변수를 분석하기.
  4. 모델이 만드는 에러의 종류를 분석하기.
  5. 간단한 특성 선택과 특성 공학 단계를 수행합니다.
  6. 이전 다섯 단계를 한 번이나 두 번 빠르게 반복하기
  7. 다른 종류의 모델을 중심으로 가장 가능성이 높은 모델을 세 개에서 다섯개 정도 추리기.

6. 모델 튜닝하기

이 단계에서는 가능한 많은 데이터 이용하기. 1. 교차 검증을 사용해 하이퍼파라미터를 튜닝하기. 2. 앙상블/배깅/스태킹 시도해보기. 3. 최종 모델이 확신이 서면 테스트 세트 이용해서 성능 측정하기.

7. 솔류션 출시

  1. 지금까지의 작업을 문서화 하기.
  2. 멋진 발표 자료 만들기 -먼저 큰 그림(스토리) 이야기하기
  3. 어떻게 문제를 해결할 수 있는지 설명하기.
  4. 작업 과정에서 알게 된 흥미로운 점들을 꼭 설명하기
    • 성공한 것과 실패한것 모두 설명
    • 우리가 세운 가정과 시스템의 제약을 나열하기
  5. 마지막은 기억하기 쉬운 문장으로 핵심 내용을 전달하기. (ex : 이런 시스템을 통해 실시간으로 악성 유저를 n% 탐지할 수 있습니다.)

8. 서비스 출시하기

  1. 서비스로 출시하기 위해 여러가지 장치들 준비하기.
  2. 시스템의 서비스 성능을 일정한 간격으로 확인하고 성능이 감소됐을 때 알림을 받기 위한 모니터링 코드 작성하기.
  3. 정기적으로 새로운 데이터에서 모델을 다시 훈련시키기
